# TODO & Changes Log

## Changes from Original Plan

### Section: Font Choice

**Location:** `scripts/build.ts`, `src/style.css`

**Original Plan:** Segoe UI for UI and code blocks

**Change:** Used Inter font (from `@fontsource/inter`) for OG images due to Segoe UI not being available as an NPM package.

**Reason:** Segoe UI is a Windows system font and not available as an NPM package. Inter is a similar clean sans-serif font that works cross-platform.

---

### Section: Frontmatter Delimiter

**Location:** `scripts/build.ts` line 56

**Original Plan:** Standard YAML frontmatter with `---` delimiters

**Change:** Modified regex to accept both `---` (YAML) and `+++` (TOML) delimiters: `/^[-+]{3}\n([\s\S]*?)\n[-+]{3}\n([\s\S]*)$/`

**Reason:** The sample posts were written with `+++` TOML-style delimiters. Updated the parser to be flexible and accept either format.

---

### Section: Date Format

**Location:** `public/posts/*.md`

**Original Plan:** Date strings like `date = 2026-01-13`

**Change:** Updated to ISO 8601 format: `date = 2026-01-13T00:00:00Z`

**Reason:** TOML requires strict date-time format. Simple date strings like `2026-01-13` caused `SyntaxError: Expected "T" but "\n" found`. Updated all post frontmatter to use full ISO 8601 format.

---

### Section: Package Substitutions

**Location:** `package.json`, `scripts/build.ts`

**Original Plan:** `@resvg/resvg-js` for OG image generation

**Change:** Initially installed `resvg` (wrong package), then switched to `@resvg/resvg-js`

**Reason:** `resvg` on NPM is a React component library, not the SVG-to-PNG renderer needed for OG images. Had to remove and reinstall the correct package.

---

### Section: Development Server Configuration

**Location:** `src/main.ts`

**Original Plan:** Assume posts.json exists in public/ at runtime

**Change:** Added async initialization that waits for posts.json to be fetched

**Reason:** posts.json is generated by the build script and placed in dist/. For dev mode to work, posts.json must also exist in public/ so Vite serves it during development.

**Fix:** Run `bun run posts` then copy `dist/posts.json` to `public/posts.json` for development.

---

### Section: Tailwind Typography

**Location:** `src/style.css`

**Original Plan:** Use Tailwind CSS v4 prose classes

**Change:** Added `@plugin "@tailwindcss/typography";`

**Reason:** The prose classes (`prose`, `prose-zinc`, etc.) are part of the official Tailwind Typography plugin and need to be explicitly enabled in Tailwind v4.
